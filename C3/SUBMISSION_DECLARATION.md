# C3赛道参赛作品声明

## 1. 原创性声明

本参赛作品为队伍独立开发，所有代码、设计方案和文档均为原创。

## 2. 第三方依赖声明

### 2.1 使用的第三方库

| 库名称   | 版本     | 用途           | 来源                   |
| ----- | ------ | ------------ | -------------------- |
| CuPy  | >=14.0 | GPU数值计算      | <https://cupy.dev/>  |
| NumPy | >=1.24 | CPU数值计算、数据处理 | <https://numpy.org/> |
| ONNX  | >=1.15 | ONNX模型解析     | <https://onnx.ai/>   |
| SciPy | >=1.11 | 特殊函数计算（erf）  | <https://scipy.org/> |

### 2.2 开源项目参考

本项目参考了以下开源项目的设计理念：

- ONNX Runtime: 参考了ONNX模型加载和算子执行的架构设计
- CuPy: 参考了GPU内存管理和算子实现的最佳实践

## 3. AI辅助生成代码声明

### 3.1 AI辅助生成的代码

本项目部分代码使用AI辅助生成，具体包括：

| 文件                                 | AI辅助内容                              | 验证方式                 |
| ---------------------------------- | ----------------------------------- | -------------------- |
| `kernels/cupy_kernels.py`          | 算子实现的初始版本（Add, Mul, Div, Relu等基础算子） | 通过与PyTorch参考输出对比验证精度 |
| `scheduler/runtime/cupy_engine.py` | 模型加载和执行框架的初始版本                      | 通过端到端测试验证功能正确性       |
| `scheduler/memory/memory_pool.py`  | 内存池和内存管理组件的初始版本                     | 通过代码审查和功能测试验证        |
| `cli/infer_worker.py`              | Worker协议实现的初始版本                     | 通过协议测试验证正确性          |

### 3.2 AI辅助生成的修改

以下修改使用AI辅助完成：

- 将C3.4内存规划组件集成到C3.5运行时
- 修复`DeviceMemoryPool`与真实GPU内存分配的挂钩问题
- 将基于文件名的batch\_size调整改为基于模型权重大小的动态策略

### 3.3 人工审查和修改

所有AI辅助生成的代码均经过人工审查和修改，确保：

- 符合竞赛规则要求
- 代码正确性和鲁棒性
- 无硬编码、无针对性优化
- 所有输出由通用算法逻辑产生

## 4. 代码完整性声明

本提交包含完整的源代码，所有产物均由提交的源码在评测环境中实时生成，不包含任何预计算的网表、预编译的二进制或其他预生成的结果文件。
